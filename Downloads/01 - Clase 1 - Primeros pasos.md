# Curso: Arch Linux e Hyprland desde Cero

# Clase 1 -- Primeros pasos

## Objetivo

Comprender la estructura básica de Arch Linux y comenzar a trabajar con
Hyprland entendiendo cada paso.

------------------------------------------------------------------------

## Lo que hicimos

-   Instalamos Arch Linux con `archinstall`.
-   Elegimos **GRUB** como gestor de arranque.
-   Elegimos el perfil **Hyprland**.
-   Configuramos **PipeWire** para el audio.
-   Configuramos **NetworkManager** para la red.
-   No instalamos paquetes adicionales para mantener el sistema limpio.

------------------------------------------------------------------------

## Primer cambio en Hyprland

Encontramos la línea:

``` lua
hl.config({ autogenerated = true })
```

La eliminamos para quitar el aviso de configuración de ejemplo.

Después recargamos Hyprland:

``` bash
hyprctl reload
```

### Tu carpeta personal

    /home/alita

aplicaciones.

## Pacman

Pacman es el gestor de paquetes de Arch Linux.

Comandos aprendidos:

Buscar paquetes:

``` bash
pacman -Ss nombre
```

Ver información:

``` bash
pacman -Si nombre
```

Ver si está instalado:

``` bash
pacman -Qs nombre
```

Instalar:

``` bash
sudo pacman -S nombre
```

Actualizar todo:

``` bash
sudo pacman -Syu
```

------------------------------------------------------------------------

## Navegación

Ver ubicación actual:

``` bash
pwd
```

Listar archivos:

``` bash
ls
```

Listar también archivos ocultos:

``` bash
ls -la
```

Entrar a una carpeta:

``` bash
cd carpeta
```

Volver a tu carpeta personal:

``` bash
cd ~
```

------------------------------------------------------------------------

## Errores aprendidos

### Error

``` text
which: command not found
```

En Arch Linux `which` puede no venir instalado.

Alternativa:

``` bash
command -v firefox
```

------------------------------------------------------------------------

### Error

``` text
bash: /: Is a directory
```

Ocurre porque `/` es una carpeta, no un comando.

Correcto:

``` bash
cd /
```

## Filosofía del proyecto

Antes de modificar cualquier cosa nos preguntaremos:

1.  ¿Qué es?
2.  ¿Dónde está?
3.  ¿Qué hace?
4.  ¿Cómo se modifica?
5.  ¿Cómo puedo volver atrás?

Nuestro objetivo no es copiar configuraciones, sino construir un
escritorio propio entendiendo cada componente.
